Bayesian RTM inversion: Fortran implementation

Usage

invert.fast(modname, observed, inits, cons, pmu, psd, plog, minp, ngibbs)

Arguments

modname
Name of the model to invert (character). Refer to `data/model.list.csv`
observed
Observed reflectance. Can be a vector, matrix, or data.frame; BUT NOTE: all are coerced via "as.matrix"
inits
Named vector of parameters to invert. Names are required!
cons
Named vector of constants. Names are required!
pmu
Numeric vector of prior means for inversion parameters. Must line up with `inits`
psd
Numeric vector of prior standard deviations for inversion parameters. Must line up with `inits`.
plog
Logical vector. Whether or not to use lognormal distribution for the prior. NOTE: `pmu` and `psd` are the distribution parameters, NOT the distribution's actual mean and standard deviation.
minp
Numeric vector of minimum values for parameters.
ngibbs
Number of iterations for MCMC

Value

Matrix (ngibbs x (length(inits) + 1)) of MCMC samples of parameters.

Details

Performs a Bayesian inversion of a Radiative Transfer Model. Sampling is performed using an adaptive Metropolis-Hastings algorithm operating independently on each parameter. The model takes both paramters and constants as an argument -- parameters (`inits`) are sampled and treated as random (unknown) while constants (`cons`) are fixed. Normal or log-normal priors are be specified by passing a vector of means (`pmu`), standard deviations (`psd`), and a logical of whether or not to use the lognorml (`plog`). Minimum values for the parameters must be given as a vector (`minp`).
